System Documentation & User Guide

Introduction

This scheduling application uses a mathematically weighted constraint-satisfaction algorithm to generate balanced game schedules for multi-table sports or gaming events. It optimizes for skill matching, rest cycles, and uniquely balances full round-robin coverage.

Quick Start Instructions

  1. Enter the total number of participating teams in the Number of Teams (X) field.
  2. Define how many games each team should play during the event. For a complete standard single round robin, set this value to exactly Number of Teams minus 1.
  3. Enter how many physical playing fields, courts, or tables are open in the Number of Tables (T) field.
  4. Click Generate Schedule to evaluate metrics, construct rounds, and view precise timing parameters.
  5. Use the Download as CSV button to save the layout directly into Microsoft Excel or Google Sheets.

Understanding the Round Robin Fix

In previous versions, the algorithm used purely probabilistic modeling based on team skill ranking. This caused a bug where teams frequently rematched close-ranked opponents before playing other teams in the pool.

The new Strict Round Robin Mode alters the generator logic. When it sits at the pairing step, it checks historical pairings. If two teams have already played against each other, they are completely excluded from matching until no other valid unplayed alternative matchups remain. This guarantees a broad, non-repetitive distribution of opponents.

Advanced Settings Decoded

Algorithm Tuning

  • Ranking Tolerance (k): Dictates the influence of team ranks. Ranks default directly to team numbers (Team 1 = Rank 1, Team 2 = Rank 2). A low value restricts matchmaking exclusively to close rank neighbors. A high value lets distant ranks match easily.
  • Consecutive Penalty: Governs player fatigue. If a team is active in Round 1, the system applies this divisor to their selection weight for Round 2, ensuring teams get a chance to sit out and rest when table capacity allows.

How the Multi-Tiered ETA Calculator Works

Rather than using a crude estimate, the timeline is calculated using real tournament variables:

  • Total Playtime: Computed as Rounds generated × Matches Per Game × Match Duration. For example, 3 matches of 2 minutes each equals 6 minutes of strict round play.
  • Scoring and Reset Time: Appends administrative minutes directly to every round block to account for writing scores down, resetting equipment, and rotating new teams to the tables.
  • Buffers & Meetings: A flat addition of minutes allocated for orientation meetings, organizer delays, and unexpected venue hiccups.
  • Random Delay Chance: Runs a probability simulation across every round generated. If a round triggers a delay, an extra 30 seconds is added to simulate live game overtimes or slow team swap-ins.
